javascript:window.open()代替window.showModelDialog()进行子页面和父页面交互

  • 当升级一个老项目时候发现在chrome中window.showModelDialog()不能用了，原来是在 chrome 37版本以后chrome正式废弃showModelDialog方法。

　　在应用showModalDialog的地方，通常是为了子页面和父页面之间进行传递信息。因为项目中使用showModalDialog函数的地方特别多，所以必须用一个类似方法实现功能，也就是window.open()

　　首先是判断子窗口关闭，因为项目是在子窗口关闭（确定按钮）的时候向父页面传递值的。

  • 父窗口:

    <head>
        <meta http-equiv=”Content-Type” content=”text/html; charset=utf-8” />
        <title>父页面</title>
        <script type=”text/javascript”>
            var w;//open 窗口对象  
            var wTimer;//计时器变量, 监听窗口关闭  
            function OpenChildPage(){
                url     = “children.html“;
                    var iWidth=400; //弹出窗口的宽度;
                var iHeight=300; //弹出窗口的高度;
                var iTop = (window.screen.availHeight-30-iHeight)/2; //获得窗口的垂直位置;
                var iLeft = (window.screen.availWidth-10-iWidth)/2; //获得窗口的水平位置;
                    w = window.open(url,window,“height=“+iHeight+“, width=“+iWidth+“, top=“+iTop+“,left=“+iLeft+“,modal=yes“);
                    if (w){
                    window.w.focus();    //判断窗口是否打开,如果打开,窗口前置
                        }
                wTimer    =window.setInterval(“wisclosed()“,500); //
                }
                function wisclosed(){
                    if(w.closed){
                    alert(window.returnTo);    //子窗体返回值  
                        window.clearInterval(wTimer);
                }

            }

question：

　　Uncaught SecurityError: Blocked a frame with origin “null” from accessing a frame with origin “null”. Protocols, domains, and ports must match.

　　跨页面操作涉及域的概念（origin）。
　　错误的意思是： 未捕获的安全错误：阻止了一个域为null的frame页面访问另一个域为null的页面。
　　原因是：         本地直接用浏览器打开的的页面吧
　　解决办法：      在本地架设服务器来调试，建议工具：nginx。

function setImagePreview() {
    var docObj = document.getElementById(“doc“);
    var imgObjPreview = document.getElementById(“preview“);
    if (docObj.files && docObj.files[0]) {
    /火狐下，直接设img属性/

    alert(“您上传的图片格式不正确，请重新选择!“);
return false;
    }
    imgObjPreview.style.display = ‘none‘;
    document.selection.empty();
    }
    return true;
