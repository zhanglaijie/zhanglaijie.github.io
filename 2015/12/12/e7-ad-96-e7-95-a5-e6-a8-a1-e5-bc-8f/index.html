设计模式

策略模式定义：定义了算法族，分别封装起来，让它们之间可以互相替换，此模式让算法的变化独立于使用算法的客户。

设计原则：找出应用中可能需要变化之处，把它们独立出来，不要和那些不需要变化的代码混在一起。
为什么要用设计模式

  1. 我们使用继承的方式进行扩展的时候如果在父类加上任意一个需要变化的方法，在子类的继承下需要分别实现。对代码的局部修改影响了不止是局部。

  2. 我们要使用接口的话当需要该方法的类实现该接口，如果对每个子类都需要稍微修改一下。代码的重复的会特别多。
    为什么有部分代码需要一直改变呢

  3. 顾客需要新功能，进行扩展

  4. web开发需要采用其他数据库产品，格式不兼容
    我们用一个案例进行分析，古代进行发动战争，将军需要用宝剑杀人，杀手需要用匕首杀人。士兵需要用弓箭杀人。
  • 如果使用继承Charactor（角色）的情况，Charactor有一个fight()抽象方法，各种角色的士兵继承父类的方法，有些人不需要fight如平民。如果需要在父类增加谋略方法的话，子类都有了这个方法，不需要的还要置空，因为士兵只需要执行命令不需要谋略。
  • 同样我们采用接口的方法 需要的实现接口不需要的可以不实现。但是如果其中一些实现接口的需要少量修改一部分内容，我们发现需要更该每个方法，重复造成了浪费。
    只有这个时候我们才发现设计模式的重要性，我们可以把两者相结合，经常需要变化的抽象并封装起来，和其他稳定的代码有所区别，代码变化引起来的不经意后果变少，系统变得更加有弹性。

我们需要把角色的经常变化的行为从Character中抽象出来。建立一组新类来代表每个行为。行为最好可以动态的变化 ，所以我们要针对接口编程，而不是针对实现编程。

我们用接口代表行为，Fighthavior接口，而行为的实现实现本接口。当然我们不能用角色实现Fighthavior接口。我们需要制造一组其他类来实现weaponBehavior 接口。AxeBehavior 和knifeBehavior 分别进行实现。

现在在Character类中加入实例变量，声明为接口而不是具体类实现类型，每个角色都会动态的设置变量意在运行的时候采用正确的行为类型，我们用一个相似的方法 performFight（）来实现行为。

类为这样，

public abstract class Character {
    WeaponBehavior weaponBehavior;

public void preformKill(){
weaponBehavior.useWeapon();
}

}

而他的子类这样，在构造函数中给父类的变量进行实现，这叫做针对超类编程

public  class Queen extends Character{
    public Queen(){
        weaponBehavior = new KnifeBehavior();
    }
}

加入 fight接口

public interface WeaponBehavior {
    public void useWeapon();
}

和他的两个实现

public class KnifeBehavior implements WeaponBehavior{ 

@Override
public void useWeapon() {
    // TODO Auto-generated method stub
    System.out.println("knife");
}

}

public class AxeBehavior implements WeaponBehavior { 

@Override
public void useWeapon() {
    // TODO Auto-generated method stub
    System.out.println("Axe");
}

}

进行测试

public class test{

public static void main(String[] args) {
Character character = new Queen();
character.setWeaponBehavior(new AxeBehavior());
character.preformKill();
}

}

可看出输出Axe

我们还可以在

character类中加入setter getter 方法

public abstract class Character {
WeaponBehavior weaponBehavior;
public WeaponBehavior getWeaponBehavior() {
return weaponBehavior;
}
public void setWeaponBehavior(WeaponBehavior weaponBehavior) {
this.weaponBehavior = weaponBehavior;
}
public void preformKill(){
weaponBehavior.useWeapon();
}
}

这样我们就可以在 运行的时候随意改变武器

public static void main(String[] args) {
        Character character = new Queen();
        character.setWeaponBehavior(new AxeBehavior());
        character.preformKill();
    }
